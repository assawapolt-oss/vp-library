VP Library Metadata System

ระบบ metadata สำหรับคลัง Virtual Production ที่มีฉาก 1,000+ รายการ ออกแบบให้ทีมงานค้นหาฉากที่ใช่ได้ภายใน 30 วินาที โดยไม่ต้องไล่ดูทีละไฟล์

หลักการออกแบบ: ยิ่ง metadata ชัด ยิ่งค้นหาได้เร็ว — แต่ถ้า tag เยอะเกินไปก็ยุ่งยากในการ maintain ระบบนี้เลือก 7 dimensions หลัก ที่ครอบคลุมความต้องการของทีม VP โดยเฉพาะ

ปัญหาที่ระบบนี้แก้

❌ ก่อนมีระบบ
• ไล่ดูไฟล์ทีละตัว
• จำชื่อไฟล์ไม่ได้
• ทีมใหม่ไม่รู้ว่ามีฉากอะไร
• ใช้ฉากซ้ำโดยไม่รู้ตัว
✅ หลังมีระบบ
• ค้นหาด้วย keyword/tag
• Filter หลาย dimension พร้อมกัน
• เห็น thumbnail + metadata ทันที
• Track การใช้งานได้

Metadata Schema — โครงสร้างหลัก

แต่ละ scene จะมี metadata ในรูปแบบนี้ ใช้เป็น JSON หรือแถวใน spreadsheet ก็ได้

// VP Scene Metadata Schema v1.0
{
  // ── IDENTITY ──────────────────────────────────
  "id": "VP-0042",                    // รหัส unique
  "filename": "sci-fi-lab-portal.mp4",
  "thumbnail": "VP-0042_thumb.jpg",
  "title": "Sci-Fi Lab with Portal Ring",
  "added_date": "2026-03-17",

  // ── ENVIRONMENT ───────────────────────────────
  "environment": "indoor",
  "location_type": "sci-fi-lab",
  "location_country": "fictional",

  // ── MOOD & TONE ───────────────────────────────
  "mood": ["dramatic", "mysterious"],
  "color_temp": "warm",              // warm | cool | neutral | mixed
  "dominant_colors": ["#8B5E3C", "#1A1A2E", "#F5A623"],
  "brightness": "dark",               // bright | medium | dark

  // ── SUBJECT & CONTEXT ─────────────────────────
  "topics": ["technology", "AI", "innovation"],
  "editorial_fit": ["Tech", "Entertainment"],

  // ── LIGHTING ──────────────────────────────────
  "time_of_day": "night",              // day | golden-hour | night | timeless
  "lighting_style": "dramatic",        // natural | studio | dramatic | neon
  "has_sfx_light": true,              // มี light effects เช่น ไฟกะพริบ, ลำแสง

  // ── MOTION & STYLE ────────────────────────────
  "motion_type": "subtle",            // static | subtle | animated | looping
  "style": "cinematic",             // cinematic | realistic | stylized | abstract
  "has_particles": true,             // ควัน, ฝุ่น, iskra ฯลฯ
  "depth_of_field": "deep",          // shallow | medium | deep

  // ── USAGE TYPE ────────────────────────────────
  "usage_type": ["standup", "interview"],
  "subject_placement": "center",     // center | left | right | flexible
  "background_complexity": "complex", // clean | medium | complex
  "platform_fit": ["16:9", "9:16"],  // aspect ratios ที่เหมาะ

  // ── TECHNICAL ─────────────────────────────────
  "resolution": "4K",
  "duration_sec": 30,
  "is_loopable": true,
  "engine": "Unreal Engine 5",
  "file_format": "mp4",
  "file_size_mb": 285,

  // ── ADMIN ─────────────────────────────────────
  "used_in": ["EP-042", "EP-051"],    // episodes ที่เคยใช้
  "notes": "ระวัง: ลำแสงสว่างมาก ต้องปรับ exposure presenter",
  "status": "active"               // active | archived | review
}

🌍 Dimension 1 — Environment & Location

มิตินี้ตอบคำถาม "ฉากนี้อยู่ที่ไหน?" เป็น dimension แรกที่ทีมงานมักค้นหา

🏠
Environment
environment
Indoor หรือ Outdoor — ส่งผลต่อการ match กับบรรยากาศรายการ
indoor outdoor space abstract
📍
Location Type
location_type
ประเภทสถานที่ ใช้ค้นหาตาม "บรรยากาศ" ที่ต้องการ
studio-newsstudio-talk sci-fi-labsci-fi-city robot-factorycity-bangkok city-genericnature-forest nature-mountaintunnel-cave warehousespace-nebula abstract-particlesfinance-trading hospitallifestyle-home
🌏
Location Country
location_country
ถ้าฉากเป็น location จริง ระบุประเทศ/เมือง สำหรับรายงานข่าวต่างประเทศ
thailandusa chinajapan europemiddle-east fictionalgeneric

🎨 Dimension 2 — Mood & Tone

สิ่งที่คนค้นหาบ่อยที่สุด — "ฉากที่ให้ความรู้สึกหนักแน่น" หรือ "ฉากสดใส สำหรับ lifestyle"

😌
Mood
mood
ระบุได้หลาย mood (array) เพราะฉากหนึ่งอาจให้ความรู้สึกหลายแบบ
dramatictense warmhopeful seriousauthoritative mysteriousfuturistic freshcheerful neutralminimal romanticcalming
🌡️
Color Temperature
color_temp
อุณหภูมิสี — สำคัญมากสำหรับ match กับ presenter skin tone และ grade ของรายการ
warm cool neutral mixed
☀️
Brightness
brightness
ความสว่างโดยรวม — กำหนดว่าต้อง light presenter อย่างไร
bright medium dark

📰 Dimension 3 — Subject & Editorial Context

เชื่อม VP scene เข้ากับ content pillar ของ THE STANDARD โดยตรง ทำให้ Producer ค้นหา "ฉากสำหรับข่าวการเมือง" ได้ทันที

🗂️
Editorial Fit
editorial_fit
Match กับ content pillars ของ THE STANDARD — ฉากหนึ่งอาจใช้ได้หลาย pillar
News Business Politics Tech Lifestyle Entertainment Climate Society
🔍
Topic Keywords
topics
Keywords เฉพาะเจาะจง สำหรับค้นหาแบบ free-text เช่น "robot", "election", "stock"
AIrobot electionparliament stockeconomy warconflict natureclimate healthmedical fashionfood

💡 Dimension 4 — Lighting

สำคัญมากสำหรับทีมกล้อง — ต้อง match แสง presenter กับ background ให้ดูเป็นธรรมชาติ

🕐
Time of Day
time_of_day
บอกโทนแสงโดยรวม — ช่วยทีมกล้องวางแผน lighting ได้เลย
day golden-hour night timeless interior-lit
🔦
Lighting Style
lighting_style
สไตล์แสงในฉาก — ส่งผลต่อว่าต้อง key light presenter แบบไหน
natural-soft studio-flat dramatic-hard neon-colored backlit practicals-only
✨
Special FX Lights
has_sfx_light
มี light effects ที่ต้องการความสนใจพิเศษหรือเปล่า เช่น ลำแสง, ไฟกะพริบ, พอร์ทัล
portal-glow light-rays flicker neon-glow none

🎬 Dimension 5 — Motion & Visual Style

บางรายการต้องการ background นิ่งๆ บางรายการต้องการ dynamic — ระบุให้ชัดเพื่อเลือกได้เร็ว

🎥
Motion Type
motion_type
ระดับการเคลื่อนไหวของฉาก — สำคัญสำหรับรายการสัมภาษณ์ที่ต้องการ background เงียบ
static subtle-ambient animated-loop high-energy camera-move
🖼️
Visual Style
style
ความเป็น cinematic vs realistic vs graphic — กำหนด "feel" ของ production
cinematic realistic stylized abstract minimal-clean documentary
🌫️
Particles & FX
has_particles
มีควัน, ฝุ่น, iskra, เมฆ, ฝน หรือ atmospheric effects
smoke dust rain sparks energy-field fog-mist none

🎯 Dimension 6 — Usage Type

บอกว่าฉากนี้เหมาะกับการถ่ายทำแบบไหน — ช่วย Producer เลือกได้ตรงตามที่ต้องการทันที

🎤
Usage Type
usage_type
รูปแบบการใช้งานหลัก — ฉากหนึ่งอาจใช้ได้หลายแบบ
standup-report interview-single interview-dual news-anchor explainer podcast-video documentary social-short
👤
Subject Placement
subject_placement
ตำแหน่ง presenter ที่เหมาะสมในฉาก — ช่วย Director of Photography วาง shot
center left right flexible wide-2-person
📱
Platform Fit
platform_fit
Aspect ratio ที่ฉากนี้รองรับ — สำคัญมากสำหรับ repurpose content
16:9 — YouTube/Web 9:16 — TikTok/Reels 1:1 — Instagram 4:3 — Legacy
🧱
Background Complexity
background_complexity
ความซับซ้อนของพื้นหลัง — ฉาก clean เหมาะกับรายการข่าว ฉาก complex เหมาะ สารคดี
clean-minimal medium complex-detailed

📁 โครงสร้างโฟลเดอร์ที่แนะนำ

ออกแบบให้ค้นหาด้วย folder browsing ได้ระดับนึง และใช้คู่กับ metadata database เพื่อ search ขั้นสูง

VP_Library/ ├── _database/ // metadata files │ ├── scenes.json // metadata ทุก scene │ ├── scenes.csv // สำหรับ Excel/Sheets │ └── taxonomy.json // tag list ที่ approved │ ├── _thumbnails/ // thumbnails ทุก scene (jpg, 400px wide) │ ├── indoor/ │ ├── sci-fi/ │ ├── studio/ │ ├── industrial/ │ ├── office/ │ └── abstract/ │ ├── outdoor/ │ ├── city/ │ │ ├── bangkok/ │ │ └── international/ │ ├── nature/ │ └── landmark/ │ ├── space/ ├── abstract/ └── _archived/ // ฉากที่เลิกใช้แล้ว
⚠️ หลักการตั้งชื่อไฟล์: ใช้รูปแบบ VP-[ID]-[location]-[mood]-[version].mp4
ตัวอย่าง: VP-0042-sci-fi-lab-dramatic-v2.mp4
ห้ามใช้ space, ตัวอักษรพิเศษ, หรือภาษาไทยในชื่อไฟล์

🔍 ระบบค้นหา — 3 ระดับ

ออกแบบ search ให้รองรับ 3 ระดับความซับซ้อน ตามทักษะและความต้องการของผู้ใช้

Level 1 — Quick Filter (ผู้ใช้ทั่วไป)

Step 1

YouTube / TikTok / IG
→

Level 2 — Advanced Search (Producer / Director)

  "background_complexity": {"not": "complex-detailed"}

Level 3 — AI-Powered Search (อนาคต)

🤖 Natural Language Search: พิมพ์ว่า "ฉากข่าวการเมืองแบบหนักแน่น สีเข้ม มี motion นิดหน่อย ใช้กับ TikTok ได้" แล้วให้ AI แปลงเป็น filter query โดยอัตโนมัติ — ทำได้ถ้ามี embedding ของ metadata
